# 前端 Vue3 项目 AI 生成代码规范（强制严格遵守）
整体前端项目代码均在frontend/web下
## 一、文件与目录规范（强制）
1. **文件命名**：统一使用**短横线命名法（kebab-case）**，如：user-info.vue、request.js、login-form.vue
2. **目录结构**：严格遵循项目标准目录，禁止随意创建目录
   - src/pages：页面组件（路由页面）
   - src/components：公共组件/业务组件
   - src/hooks：自定义钩子
   - src/api：接口请求
   - src/utils：工具函数
   - src/store：状态管理（Pinia）
   - src/styles：全局样式
3. **单文件职责**：一个文件只做一件事，禁止大杂烩代码
4. **组件拆分**：超过 300 行的组件必须拆分子组件

## 二、Vue3 代码编写规范（强制）
### 1. 语法规范
1. 统一使用 **Vue3 组合式API + `<script setup>`**，禁止使用选项式API
2. 变量定义规范：
   - 基础数据：`const` 定义（禁止var）
   - 响应式数据：简单类型用 `ref`，引用类型用 `reactive`
3. 代码顺序固定（必须按此顺序编写）：
   1. 引入依赖（import）
   2. 定义props / defineEmits
   3. 定义响应式数据
   4. 定义计算属性computed
   5. 定义生命周期钩子
   6. 定义业务方法
   7. 模板template

### 2. Props / 事件规范
1. Props 必须**注释说明 + 默认值**，必填项标注
2. 事件使用 `defineEmits` 严格定义，禁止无声明的自定义事件
3. 禁止直接修改 props，必须通过 emit 通知父组件

### 3. 模板 template 规范
1. 组件必须有**唯一根节点**
2. 自定义组件使用 **短横线命名**
3. 事件绑定：`@click="handleXxx"` 方法必须以 `handle` 开头
4. 条件渲染：优先 v-if，频繁切换使用 v-show
5. 列表渲染：`v-for` 必须绑定 `:key`，key 使用唯一ID，禁止用index
6. 不同功能区域的html标签内容增加相关的注释进行说明,便于后续AI快速了解文件

## 三、JavaScript 规范（强制）
1. 禁止使用 `var`，统一使用 `const`/`let`
2. 常量命名使用**大写+下划线**：`const MAX_COUNT = 10`
3. 方法命名：小驼峰 `getUserList`、`handleSubmit`
4. 禁止魔法数字/字符串，必须抽离为常量并注释
5. 异步方法统一使用 `async/await`

## 四、注释规范（强制，对齐Python标准）
### 1. 文件注释（必选）
每个.vue / .js 文件顶部必须加文件注释：
```js
/*
 * @Author: 项目规范
 * @Date: 自动生成
 * @Description: 文件功能说明（必须写清楚）
 */
```

### 2. 组件注释（强制）
```js
/*
 * 组件功能：XXX页面/组件
 * 业务说明：负责XXX功能，包含XXX逻辑
 */
```

### 3. 变量/数据注释（强制）
1. 所有响应式变量、普通变量必须加行注释
2. 接口入参、出参必须标注字段含义
3. 常量、状态标识必须注释含义
4. 对象/数据结构必须标注每个字段含义

```js
// 用户信息数据模型
const userInfo = reactive({
  id: 0, // 用户ID
  name: '', // 用户姓名
  phone: '' // 手机号
})

// 页面加载状态
const loading = ref(false)
```

### 4. 函数/方法注释（强制）
所有函数必须包含：**功能说明、入参、出参、注意事项**
```js
/**
 * 获取用户列表数据
 * @param {Object} params - 查询参数
 * @param {Number} params.page - 页码
 * @param {Number} params.size - 每页条数
 * @returns {Promise} 列表数据 + 分页总数
 */
const getUserList = async (params) => {
  // 业务逻辑...
}
```

### 5. 核心流程注释（强制）
1. 核心业务逻辑分步骤注释
2. 判断/循环必须说明条件目的
3. 接口请求、定时器、本地存储必须注释作用
4. 复杂逻辑必须逐行说明意图

```js
// ========== 步骤1：获取查询参数 ==========
// ========== 步骤2：请求接口获取列表数据 ==========
// ========== 步骤3：处理数据渲染页面 ==========
```

## 五、样式规范（强制）
1. Vue 组件样式必须加 `scoped`
2. 类名使用短横线命名：`.user-info-box`
3. 禁止行内样式，统一写在style标签
4. 公共样式抽离到全局css，禁止组件重复编写
5. 单位优先使用 rem / vw，适配项目规范

## 六、API / 请求规范（强制）
1. 所有接口放在 `src/api` 目录，按模块拆分文件
2. 接口方法必须加详细注释
3. 必须做异常捕获 `try/catch`
4. 加载状态、错误状态统一处理
5. 统一封装请求工具，禁止在组件内直接写axios

## 七、代码格式与校验（强制）
1. 缩进统一使用 **2个空格**
2. 语句末尾必须加分号
3. 字符串统一使用单引号
4. 代码自动格式化，风格统一
