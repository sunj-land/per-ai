# 个人强制代码规范：Python 注释规则（必须严格遵守）
## 1. 数据注释规范（强制）
1. 所有类必须增加文档字符串，说明类用途。
2. 所有实例变量、类变量必须增加行注释，说明含义。
3. 所有字典结构（如返回体、数据模型）必须增加键值注释。
4. 数据库字段、JSON 结构必须标注每个字段含义。
5. 常量必须注释含义。

## 2. 方法/函数注释规范（强制）
1. 所有函数/方法必须包含文档字符串，包含：功能说明、入参、出参、异常。
2. 函数内部关键分支必须加中文注释。
3. 核心业务逻辑必须逐行说明意图。
4. 工具方法、公共方法必须详细注释。

## 3. 主要流程注释规范（强制）
1. 核心业务流程必须分步骤注释（步骤1、步骤2、步骤3...）。
2. 判断逻辑、循环逻辑必须说明条件目的。
3. 数据库操作、网络请求必须注释作用。
4. 事务、批量操作必须注释说明执行逻辑。

## 4. 注释格式（固定风格）
- 类注释：
  """
  类功能说明
  """

- 函数注释：
  """
  函数功能说明
  :param 参数名: 参数说明
  :return: 返回值说明
  """

- 行内注释：
  # 注释内容

- 流程注释：
  # ========== 步骤1：XXX ==========
  # ========== 步骤2：XXX ==========

- 数据结构注释：
  # 字段名: 字段说明
